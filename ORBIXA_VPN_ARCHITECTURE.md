# Orbixa VPN: Technical Architecture & System Documentation

Orbixa VPN is a high-performance, privacy-first VPN solution designed for modern web and desktop environments. It leverages the **VLESS protocol** over **Xray-core** to provide undetectable internet tunneling with minimal overhead.

---

## 🚀 Core Philosophy
**"Secure. Private. Borderless."**
- **Undetectable:** Uses XTLS Reality and Reality-Vision to blend VPN traffic with standard HTTPS.
- **Performant:** Minimal encryption overhead compared to legacy protocols like OpenVPN.
- **User-Centric:** Simple dashboard for config generation and native desktop performance.

---

## 🛠 Tech Stack

### 1. Backend Control Plane
- **Framework:** Spring Boot (Java 17)
- **Database:** MongoDB (Atlas/Local)
- **Security:** Spring Security + JWT (JSON Web Tokens)
- **Connectivity:** JSch (Java Secure Channel) for SSH-based node management.

### 2. Frontend Web Interface
- **Framework:** Next.js 14 (App Router)
- **Styling:** TailwindCSS + Framer Motion (Animations)
- **Icons:** Lucide-react
- **Design System:** Cyberpunk/Minimalist dark theme with electric blue highlights.

### 3. Desktop Application
- **Core:** Tauri (Rust-based back-end) / Electron (Legacy Support)
- **UI:** React.js + Vite
- **Networking:** Native bindings to execute bundled Xray binaries.

### 4. VPN Server Nodes
- **Engine:** Xray-core (VLESS Protocol)
- **Automation:** Bash scripts (`deploy_node.sh`, `add_user.sh`) for rapid VPS deployment.
- **OS:** Ubuntu 22.04+ Recommended.

## 🔐 Security & Networking Model

### 1. Authentication Flow
1. User logs in via the Web Dashboard or Desktop Client.
2. Backend validates credentials/JWT and returns a **User UUID**.
3. This UUID is the unique "Key" used for VLESS authentication.

### 2. VLESS Config Generation
When a user selects a server:
- The backend generates a VLESS URI: `vless://{UUID}@{HOST}:{PORT}?type=ws&security=tls&path=/orbixa#{NAME}`.
- Simultaneously, the backend uses **SSH (SshService)** to connect to the target VPN server.
- It executes `add_user.sh` to inject the user's UUID into the server's `config.json` without downtime.

### 3. Usage Tracking
- The VPN client (either mobile or desktop) periodically reports bandwidth usage to the `/api/vpn/usage` endpoint.
- Data is aggregated in MongoDB (`UsageLog` collection).
- If a user hits their `BandwidthLimit` (specified in `Subscription`), access can be revoked.

## 🖥 Admin Features

The Admin Panel (`/admin`) allows centralized management of the global infrastructure:
- **Node Deployment:** Admins can trigger the `deploy_node.sh` script which automates the installation of Xray, SSL certs via Certbot, and UFW firewall configuration on a fresh VPS.
- **User Monitoring:** Real-time visibility into registered users, their subscription status (FREE vs PRO), and system activity.
- **Server Health:** Monitor online/offline status of high-speed global nodes.

## 📦 Deployment Guide

### Local Development (Docker)
1. Ensure Docker and WSL2/Linux are active.
2. Run `docker-compose up --build`.
3. Access Frontend: `http://localhost:3000`.
4. Access Backend: `http://localhost:8080`.

### Production Server Deployment
1. Provision a fresh Ubuntu 22.04 VPS.
2. Run the `deploy_node.sh` command generated by the Admin Dashboard.
3. Configure the Domain/Email when prompted for SSL.
4. Add the Node IP/Domain to the Admin Dashboard.
